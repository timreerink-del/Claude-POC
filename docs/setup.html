Step-by-step setup

1
Install Node.js 20 via nvm 
nvm install 20 && nvm use 20

A .nvmrc file is included in the repo — the correct version will be picked up automatically.

2
Clone the repo and run the setup script 
git clone https://github.com/timreerink-del/Claude-POC.git
cd Claude-POC/dmp-prototype
./setup/team-setup.sh

The setup script installs npm dependencies, Expo CLI, and Claude Code CLI, then runs a health check.

3
Install Expo Go on your phone

Download from the App Store or Google Play. Used for testing on a real device.

4
Sign in to Claude Code 
claude login

Requires an Anthropic account. Sign up at claude.ai first if needed.

5
Request Figma access

Ask the project owner to invite you to the Figma file:

🔗 figma.com/design/amLMUl0QJ1LCFErsiE4hkg/DMP-x-Claude-POC
6
Generate your Figma personal access token

Go to your Figma account settings and create a token:

🔗 figma.com/developers/api#access-tokens

Copy it — you'll need it in the next step.

7
Install the Figma MCP (Claude Code + Figma bridge) 
claude mcp add figma-console -s user \
  -e FIGMA_ACCESS_TOKEN=figd_YOUR_TOKEN_HERE \
  -e ENABLE_MCP_APPS=true \
  -- npx -y figma-console-mcp@latest

Replace figd_YOUR_TOKEN_HERE with your token from step 6.

Daily workflow

  1. Pull latest: git pull
  2. Run guardian: ./scripts/guardian.sh
  3. Create restore point before new work: ./scripts/restore-point.sh "your-feature-name"
  4. Build your feature
  5. Run guardian again: ./scripts/guardian.sh
  6. Create final restore point: ./scripts/restore-point.sh "your-feature-name-complete"
  7. Push: git push

Design rules

  • Figma is always the source of truth
  • Use only design tokens — no hardcoded hex values
  • Read CLAUDE.md before starting any Claude Code session
  • Read .claude/skills/ui-ux-pro-max/SKILL.md before any UI work
